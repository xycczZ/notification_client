# 通知告警系统使用指南

## 快速开始

### 1. 启动开发服务器

```bash
npm run dev
```

开发服务器将运行在 http://localhost:1420

### 2. 启动后端服务器

确保后端服务器正在运行，默认地址为 http://localhost:3002

### 3. 访问应用

在浏览器中打开 http://localhost:1420

## 功能说明

### 起始界面

- **服务器地址**: 输入后端服务器的地址，如 `http://localhost:3002`
- **认证码**: 输入后端服务器的认证码（如有）
- **连接**: 点击按钮尝试连接到后端服务器

### 首页仪表盘

连接成功后，自动跳转到首页仪表盘，包含以下部分：

#### 统计卡片
- 总通知数
- 各类型通知数量（ERROR、CRITICAL、ALERT等）

#### 图表展示
- 显示各类型通知的数量统计图表

#### 实时通知列表
- 以聊天框形式展示实时通知
- 显示通知标题、内容、时间
- 新通知会实时推送到列表顶部

## 测试应用

### 1. 运行测试脚本

```bash
./test-app.sh
```

### 2. 手动测试

1. 打开浏览器访问 http://localhost:1420
2. 在起始界面输入后端服务器地址
3. 输入认证码（如有）
4. 点击"连接"按钮
5. 连接成功后会自动跳转到首页仪表盘

### 3. 发送测试通知

运行Python测试脚本发送测试通知：

```bash
python3 test-send-notification.py
```

## 故障排除

### 连接失败

1. 检查后端服务器是否正在运行
2. 检查服务器地址是否正确
3. 检查认证码是否正确
4. 检查网络连接是否正常

### 通知不显示

1. 检查WebSocket连接是否正常
2. 检查后端是否发送了通知
3. 检查浏览器控制台是否有错误信息

### 系统通知不工作

1. 检查系统通知权限是否已授予
2. 检查Tauri通知插件是否正确配置

## 配置说明

### 后端配置

后端默认配置：
- 端口: 3002
- 认证码: 由环境变量 `AUTH_CODE` 设置，默认为 `slkdg_aokgl`

### 前端配置

前端配置通过起始界面进行，无需额外配置文件。

## 技术支持

如有问题，请检查：
1. 控制台错误信息
2. 网络连接状态
3. 后端服务器日志
